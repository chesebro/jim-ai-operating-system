# Skills Overview

This repo contains reusable Claude skills for strategy, product, GTM, writing, and execution.

## Product / PMlls

### pm-interviewer
Use when an idea is still fuzzy and you need to clarify:
- the user
- the problem
- the workflow
- the friction
- the constraints
- what should stay human vs delegated

Best for discovery and creating a strong context doc before strategy or build work.

### pm-decision-workflow
Use when you want to evaluate what the product should actually do.

Focuses on:
- workflow design
- delegation (assist / draft / execute / monitor)
- trust model
- artifact design
- moat / defensibility
- data over time
- recommendation

Best for product strategy, roadmap, and AI-native product design.

## GTM Skills

### gtm-interviewer
Use when the GTM story is still fuzzy and you need to clarify:
- buyer
- user
- painful workflow
- trigger
- alternatives
- wedge
- GTM constraints

Best for discovery before writing strategy or messaging.

### gtm-builder
Use when you need a full GTM plan or commercialization strategy.

Focuses on:
- painful workflow
- buyer / user roles
- trigger
- wedge
- positioning direction
- motion
- channels
- roadmap
- risks

### positioning-messaging
Use when you need buyer-facing language that is sharper and more differentiated.

Focuses on:
- positioning
- category framing
- value proposition
- differentiation
- headline / subhead options
- message pillars

### gtm-experiment-designer
Use when you want to test a GTM angle quickly and intelligently.

Focuses on:
- risky assumptions
- practical experiments
- signal
- next-step logic

### sales-outbound-builder
Use when you want to turn strategy into practical outreach.

Focuses on:
- buyer targeting
- relevance hooks
- outreach angle
- sequence structure
- concise drafts
- credibility support

### gtm-critique
Use when you want a blunt assessment of whether the GTM is actually good.

Focuses on:
- pain clarity
- buyer clarity
- urgency
- differentiation
- commercial sharpness
- biggest risks

## Suggested PM Workflow

1. Start with `pm-interviewer` if the idea is still fuzzy
2. Move to `pm-decision-workflow` to decide what the product should do

## Suggested GTM Workflow

1. Start with `gtm-interviewer` if the GTM is still fuzzy
2. Use `gtm-builder` to define the commercial approach
3. Use `positioning-messaging` to sharpen buyer-facing language
4. Use `gtm-experiment-designer` to test the riskiest assumption
5. Use `sales-outbound-builder` for founder-led sales and outreach
6. Use `gtm-critique` to pressure test the whole thing
