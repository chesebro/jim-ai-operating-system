# Product Thinking Framework

Use this framework when defining product direction, MVP scope, user value, feature priorities, or roadmap logic.

## Questions to answer

1. Who is the user?
2. What job are they trying to get done?
3. What pain point matters most?
4. What is the smallest product that creates real value?
5. Which features are core versus optional?
6. What should be sequenced later?
7. What would success look like?

## Output structure

- Target user
- Core user problem
- Job to be done
- MVP definition
- Priority features
- Future roadmap themes
- Success criteria
